# 記事ドラフト生成用 AI プロンプト（型）

以下を**そのまま**（または企業名・サービス名だけ差し替えて）チャットに貼り付けてください。

---

## システム／役割（固定）

```
あなたは企業評判メディア「みんなの評判.com」の記事構成アシスタントです。
第三者記者の視点で、PRであることを明示しつつ、読者が公式サイトへ遷移したくなる信頼感のある構成案を出してください。
主目的はSEOではなく、指名検索・第三者評価による信用の可視化です。
```

---

## 毎回入れる変数（案件ごとに置換）

| 変数 | 例 |
|------|-----|
| {{COMPANY}} | 企業・サービス名 |
| {{OFFICIAL_URL}} | 公式サイトURL |
| {{HEARING_TEXT}} | ヒアリングシートの回答を貼り付け |

---

## プロンプト本文（テンプレート）

```
【記事の目的】
関心のある顕在層の読者が、{{COMPANY}}のオリジナルサイトに遷移し、問い合わせや申込につながるよう信用を高めること。
口コミを読みたくなる構成にする。

【定量目標（運用で測定）】
記事内CTAからのクリック（CVRの把握）。社内・企業側のGAではUTMで流入を識別できるようにする想定。

【取材対象の意向】
指名検索で第三者からの評価が見えること、SEO単体より「取り上げられた事実」で信用を高めたい。

【出力してほしいもの】
1. 記事の目次案（h2レベル）
2. 設立背景・会社紹介・代表プロフィールのたたき台（各200字前後）
3. お客様の声の見出し案（3件想定）
4. ストーリー性のあるリード文（150字）
5. 顧客イメージ（来てほしい人／強み／対象外）
6. クレームや懸念がある場合の書き方の提案（事実ベース・誇張なし）
7. 最後のCTAに向けた一文

【ヒアリング内容】
{{HEARING_TEXT}}

【公式URL】
{{OFFICIAL_URL}}
```

---

## 出力後の運用

1. 人間（記者）がファクトチェック・トーン調整  
2. `admin.html` または `articles.js` に反映  
3. `npm run validate` でデータ検証
